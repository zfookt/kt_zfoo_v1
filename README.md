# 综述
发现大佬们基本上都有自己的一套框架，确实如此，任何东西，如果自己没有从头到尾敲一遍，那就不能说明掌握了，不管多复杂的东西，只要自己亲自敲过一遍，并且理解了代码的含义那就会觉得不过如此。

目前把这所有的东西都放在一个文件夹下其实是不合适的，因为真正的公司开发中肯定是: 前端是拿不到后端代码，后端可以拿到前端代码，所以肯定是前端是独立的工程  像proto之类的则是前后端都是可以拿到，后续会逐步迭代为多工程版本。

部署的话也很重要，本地每个人都希望有一套环境，可以自由的调时间，这样子开发很方便，自测随便调时间而不影响它人。

线上部署则是多台机器，可能是分区分服，也可能是大服，也就是逻辑上支持分区分服，但是数据却是放到一个库中，然后使用分片技术支持海量数据，但是总体部署肯定是多台机器，而且服务器一般是一个是打包机，另外N台是业务机器，因此批量实现免密登录，且自动化部署是必要的，研究了k8s，发现太复杂了，而且有IP漂移的问题，最终还是觉得ansible靠谱，使用下来确实发现非常靠谱。
